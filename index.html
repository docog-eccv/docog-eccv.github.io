Abstract

Existing methods for Visual Document Question Answering (DocVQA) often predict answers without explicit intermediate analysis or produce ungrounded intermediate steps. The few approaches which employ grounding produce rectangular bounding boxes which fail to accurately represent complex structures such as pie-chart wedges, flowchart nodes, and curved text. We present DoCoG, a mask-based multi-type grounded step-wise Chain-of-Thought architecture for Document QA.

The model supports multi-type grounding, i.e., it localizes both textual content and fine-grained graphical elements such as legends, pie slices, and flowchart nodes. It generates step-wise intermediate outputs and final answers grounded through precise segmentation masks, enabling consistent evidence alignment. DoCoG integrates a Vision-Language Model with a promptable segmenter via a novel Grounding Interaction Module (GIM). During decoding, special [GND] tokens trigger mask prediction, linking each step and the final answer to supporting evidence. GIM refines [GND] tokens during decoding to improve spatial consistency and alignment.

We introduce DoCoG-QA, a large-scale diverse dataset comprising 325K documents and 1.5M step-wise grounded QA pairs. We further release DoCoG-PQA with 20K preference pairs to improve alignment using Direct Preference Optimization (DPO), and DoCoG-Bench, a human-verified benchmark. Experiments show that DoCoG achieves state-of-the-art performance across multiple benchmarks, improving answer accuracy and grounding precision.

Method

DoCoG architecture: Given a document image \(I\) and question \(q\), the VLM 1 encodes visual features and, through its LM head, autoregressively generates step-wise outputs and the final answer interleaved with [GND] tokens 2. The hidden states of the [GND] tokens 3 are passed to the Grounding Interaction Module (GIM) 4, which refines them using visual features 6 from segmenter 5 to produce grounding-aware prompt embeddings. These embeddings are then processed by the segmenter's prompt encoder 7 and mask decoder to generate evidence masks (2c), grounding each step in the document.

🔗 Decoupled Language & Spatial Grounding

DoCoG decouples language modeling from spatial prediction. The VLM focuses on step-wise reasoning and answer generation, while a dedicated segmentation module handles fine-grained, geometry-aware mask localization — yielding cleaner text outputs and more precise evidence masks.

🔄 Grounding Interaction Module (GIM)

GIM refines [GND] token embeddings through self-attention and cross-attention with visual features from the segmenter. This ensures grounding tokens remain mutually consistent and spatially aligned with the correct document regions, improving grounding fidelity.

Step-wise Multi-Type Grounding

DoCoG generates intermediate reasoning steps and final answers, each grounded to supporting evidence. Evidence spans both textual and graphical elements.

Training

DoCoG is trained in two stages: SFT on DoCoG-QA, then DPO on DoCoG-PQA.

𝓛total = λlm𝓛lm + λseg𝓛seg + λanchor𝓛anchor

𝓛lm — Language Modeling

Cross-entropy over all output tokens for accurate step-wise CoT generation and answer prediction.

𝓛seg — Segmentation (Dice + BCE)

Supervises mask predictions against pixel-level ground truth for each [GND] token.

𝓛anchor — Anchor Loss

Preserves [GND] token compatibility with the frozen segmenter during DPO, preventing mask degradation.

DPO — Preference Optimization

Trained on DoCoG-PQA (20K pairs) to align step quality, answer accuracy, and grounding fidelity jointly.
